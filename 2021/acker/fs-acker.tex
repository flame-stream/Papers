\begin{abstract}
Stream Processing Engines (SPEs) handle a potentially infinite sequence of data elements. This sequence often can be split into finite substreams of messages: epochs, windows, etc. Some operators ought to know the boundaries of these substreams. For instance, stateful streaming operators should clear outdated state, time window operators should release output after all elements within the specified time range arrived. Most state-of-the-art SPEs use {\em punctuations} to manage substreams. The punctuations approach is powerful but has limitations: it does not support cyclic dataflows, poorly scalable due to intensive use of broadcasts, and is inefficient for small substreams because of high network traffic overhead. In this work, we present a new framework called \tracker\ that fills the gap when these limitations become relevant. This method bases on the idea that a special agent monitors for substreams and notifies SPE nodes about substreams lifespan events. We demonstrate that our implementation can provide the same semantics of notifications as punctuations but implies lower network traffic overhead and has lower notification latency. Experiments show that our technique is scalable, efficiently handles real-world cyclic dataflows, and outperforms punctuations for small substreams.
\end{abstract}
